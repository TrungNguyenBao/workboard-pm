Code Standards

Engineering Guidelines & Best Practices — Last updated: 2026-03-03

General Principles

  • YAGNI / KISS / DRY — build what is needed, keep it simple, avoid repetition.
  • File size: keep individual files under 200 lines; split at logical boundaries.
  • File naming: kebab-case with descriptive names.
  • Error handling: always use try/catch or FastAPI exception handlers; never swallow errors silently.

Backend (Python / FastAPI)

Naming Conventions

Construct Convention Example
Files snake_case activity_log.py
Classes PascalCase ActivityLog
Functions snake_case create_activity()

Router Pattern

Routers are thin and delegate logic to services immediately:

@router.get("/projects/{id}/activity")
async def project_activity(id: UUID, db: AsyncSession = Depends(get_db)):
    return await list_activity(db, project_id=id)

Frontend (TypeScript / React)

Folder Structure

features/{name}/
  components/   # UI components
  hooks/        # TanStack Query hooks
  pages/        # Route-level page components

Component Rules

  • One component per file, name matches component in kebab-case.
  • Keep components under 200 lines.
  • No business logic in page components — delegate to hooks.

Testing

  • Backend: Use pytest with SQLite in-memory for fast integration tests.
  • Frontend: Vitest with React Testing Library; co-located in features folder.
  • E2E: Playwright for critical user flows (login, task creation).

Git & Commit Conventions

Use conventional commits: feat:, fix:, docs:, refactor:, test:, chore:.
