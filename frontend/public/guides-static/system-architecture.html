System Architecture

Core Backend, Frontend & Infrastructure — Last updated: 2026-03-07

High-Level Overview

Browser (React 18 + Vite)
        │  REST + SSE
        ▼
FastAPI (Python 3.12)
        │
        ├── Module routers  (/pms, /wms, /hrm, /crm)
        ├── Agent layer      (BaseAgent → domain stubs → orchestrator)
        ├── MCP protocol     (envelope, bus, context, policy)
        ├── PostgreSQL 15    (primary store, FTS, LISTEN/NOTIFY path available)
        ├── Redis 7          (ARQ background jobs, session cache)
        └── File storage     (local disk dev → MinIO/S3 prod)

All components run via Docker Compose in development. The SSE broker is in-process (no PostgreSQL LISTEN/NOTIFY in use yet; Redis Pub/Sub is the planned upgrade path for multi-instance).

Authentication Flow

Login via POST /auth/login returns an access_token in the response body (memory) and a refresh_token in a Secure HttpOnly cookie.

  • Access Token: Short-lived JWT.
  • Refresh Token: Long-lived, stored in database hash.
  • Security: All passwords hashed with bcrypt via passlib.

RBAC Model

Dimension Roles (low → high) Checked via
Workspace guest → member → admin require_workspace_role()
Project viewer → editor → owner require_project_role()

Real-time (SSE)

The system uses Server-Sent Events (SSE) for real-time updates. Services call a publish() function that delivers events to all active subscriber queues in a workspace-isolated manner.

Frontend Architecture

  • State: TanStack Query for server state, Zustand for client state.
  • UI: shadcn/ui components with Tailwind CSS.
  • D&D: @dnd-kit for complex Kanban board interactions.
